# Caliper CV Screening — Demo Script

**URL:** https://caliper.nextventures.io  
**Duration:** 15–20 minutes (full) · 8 minutes (fast)  
**Best demo job:** A Recruitee-linked role with criteria saved and at least one completed run (e.g. **SQA Engineer**).

---

## Pre-demo checklist

- [ ] VPN on (if required for Recruitee in your environment)
- [ ] Logged in with **recruiter** access (run screening + dispositions)
- [ ] Hard refresh the browser (Cmd+Shift+R)
- [ ] Know one **completed run ID** to open if a live run would take too long
- [ ] Optional: second user account to show **shared runs**

**Opening line:**  
“Caliper screens CVs against job-specific criteria, ranks candidates with evidence from each CV, and lets recruiters decide and sync outcomes back to Recruitee—without reading every application manually.”

---

## Full script (~18 min)

### 1. Jobs hub — 2 min

| Step | Action | Say |
|------|--------|-----|
| 1.1 | Sidebar → **Jobs** | “This is where every Recruitee position lives in Caliper.” |
| 1.2 | Scan the list: dept, applicants, runs | “We see volume and screening history at a glance.” |
| 1.3 | Click your demo job (e.g. **SQA Engineer**) | “I’ll walk through one role end to end.” |

---

### 2. Job detail — Overview & Criteria — 4 min

| Step | Action | Say |
|------|--------|-----|
| 2.1 | **Overview** tab | “Role context, applicant count, and entry point for a new screening run.” |
| 2.2 | **Criteria** tab | “Before any CV is scored, we define what ‘good’ looks like.” |
| 2.3 | Show **Must / Nice / Flag** lists | “Must-haves drive the base score; nice-to-haves and flags adjust it.” |
| 2.4 | Point at weights / biased flags if any | “Recruiters control the rubric—not a generic AI guess.” |
| 2.5 | *(If empty)* Add one criterion and **Save** | “Criteria are saved per job and reused on every run.” |

**Skip deep dive on:** Applicants, Talent, Activity tabs unless audience asks.

---

### 3. Run screening — 4 min

| Step | Action | Say |
|------|--------|-----|
| 3.1 | Click **Run screening** | “We pull CVs from Recruitee—no export/import circus.” |
| 3.2 | **Step 1 — Select CVs** | |
| 3.3 | Show **Recruitee applicants** selected | “CVs come straight from the ATS pipeline.” |
| 3.4 | Show stage chips (Applied, TA Interview, …) | “You can screen one stage or everyone.” |
| 3.5 | Note callout if shown | “Applicants already screened on this job are hidden so we don’t pay twice.” |
| 3.6 | Show selection count | “Only new applicants are in this list.” |
| 3.7 | **Continue** → **Step 2 — Review & run** | |
| 3.8 | Show CV count, criteria count, model | “You see cost estimate and model before committing.” |
| 3.9 | **Choose one:** | |
| | **A)** Click **Run** on a **small** selection (5–10 CVs) | “Scoring runs in the background—we’ll open results when ready.” |
| | **B)** **Cancel** and use an existing run (recommended for live demo) | “For today I’ll open a completed run so we see results immediately.” |

---

### 4. Processed CVs & results — 7 min

| Step | Action | Say |
|------|--------|-----|
| 4.1 | Sidebar → **Processed CVs** *(or job → **Runs** tab)* | “Every screening run across all jobs.” |
| 4.2 | Sort / filter table | “Status, date, job, score range, shared runs.” |
| 4.3 | Open a **completed** run | “This is the recruiter workspace.” |
| 4.4 | Scan ranked table: score, status, confidence | “Candidates ordered by fit to **this** job’s criteria.” |
| 4.5 | Use search or pipeline filter | “Filter by stage or disposition as you shortlist.” |
| 4.6 | **Click a candidate** | |
| 4.7 | Criteria checklist + quotes | “Every score is tied to evidence pulled from the CV.” |
| 4.8 | Open **CV viewer** | “Original document alongside the AI assessment.” |
| 4.9 | Toggle **agree / override** on one criterion *(if allowed)* | “Humans stay in the loop—overrides are auditable.” |
| 4.10 | Close detail; select 2–3 rows → **Compare** | “Side-by-side for final shortlist decisions.” |
| 4.11 | **Shortlist** or **Move to stage** on one candidate | “Decisions can push back to Recruitee.” |
| 4.12 | Confirm push modal if shown | “Stage moves sync to the ATS pipeline.” |
| 4.13 | *(Optional)* **Share run** with a teammate | “Collaboration without forwarding spreadsheets.” |

**Closing line for this section:**  
“The goal isn’t a black-box score—it’s faster, consistent screening with traceable evidence and ATS sync.”

---

### 5. Wrap-up — 2 min

| Step | Action | Say |
|------|--------|-----|
| 5.1 | **Activity Log** | “Runs, criteria edits, dispositions, and Recruitee sync—all logged.” |
| 5.2 | **Usage** *(if editor/admin)* | “Finance and ops can see AI spend per run and user.” |
| 5.3 | **Settings** *(admin only)* | “Models, confidence threshold, integrations, team access.” |

**Closing line:**  
“Questions on criteria setup, Recruitee sync, or rolling this out to more roles?”

---

## Fast script (~8 min)

1. **Jobs** → open demo job  
2. **Criteria** → 30 sec on must/nice/flag  
3. **Run screening** → show auto-hidden screened applicants → **Cancel** (don’t wait for run)  
4. **Processed CVs** → open completed run  
5. Open **top candidate** → criteria + CV quote  
6. **Compare** two candidates  
7. One **disposition** or **stage move**  
8. **Activity Log** → one sentence

## Demo dos and don’ts

| Do | Don’t |
|----|--------|
| Use a job with **saved criteria** | Demo a job with zero criteria |
| Open a **completed run** for results | Start a 500+ CV run and wait |
| Mention **hidden already-screened** applicants | Re-litigate old selection bugs |
| Show **one** override or disposition | Click through every tab |
| Hard refresh before presenting | Assume cache is fresh |

---

## FAQ — quick answers

| Question | Answer |
|----------|--------|
| Where do jobs come from? | Synced from **Recruitee**. |
| Can we upload CVs manually? | Yes, on non-Recruitee jobs or via **Upload CVs** in the run sheet. |
| Why aren’t some applicants in the run sheet? | Already screened on this job—they’re excluded to avoid duplicate cost. |
| How is scoring done? | AI evaluates CV text against saved **must / nice / flag** criteria. |
| Does it replace Recruitee? | No—it augments screening and can **push stages/dispositions** back. |
| Multi-user? | Workspace roles, **run sharing**, activity log. |
| Cost control? | Pre-run estimate + **Usage** page. |
